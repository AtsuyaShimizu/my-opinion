## 1. 改修の目的

### 1.1 問題1: 用語が直接的すぎる

現在のUIには「エコーチェンバー」「属性」「オピニオン・スペクトラム」「コンセンサス・メーター」など、学術用語やカタカナ語が多用されている。これらは：

- 初見ユーザーに堅い印象を与える
- 「属性」という語は行政的・システム的で人間味がない
- カタカナの機能名は何をするものか直感的に伝わらない

**目標**: 日本語として自然で柔らかく、意味が伝わる表現に統一する。

### 1.2 問題2: PostCard / フィードがXに酷似

現在のPostCardは以下の点でXのツイートカードと同じ構造：
- アバター + 名前 + @ハンドル + 時間 の横並びヘッダー
- その下に本文テキスト
- 下部にリプライ / リポスト / Good / Bad のアクションバー
- 左端に3px幅のカラーバーがあるが、見た目の印象はXと変わらない

**目標**: 「これはXではない」と即座にわかるビジュアルにする。「意見の立場」を最も目立たせる。

### 2.3 Good/Bad から 共感/異論 への変更ルール

**UI表示**: ユーザーに見えるすべてのテキストで「Good」→「共感」、「Bad」→「異論」に変更。

- PostCardのアイコンラベル（ツールチップ等）
- 投稿分析の「Good / Bad 比率」→「共感 / 異論 の比率」
- 投稿分析のパイチャート凡例: `Good` → `共感`, `Bad` → `異論`
- 投稿分析のクロス集計棒グラフ凡例: 同上
- スペクトラム（意見マップ）のY軸ラベル: `Good率 %` → `共感率 %`
- スペクトラムツールチップ: `Good 12` → `共感 12`, `Bad 3` → `異論 3`
- 通知設定: `Good通知` → `共感通知`
- 分析ページ: 「/ 20 評価」→ 「/ 20 リアクション」

**アイコン**: ThumbsUp / ThumbsDown はそのまま維持（直感的で分かりやすい）。

## 3. PostCard / フィード デザイン刷新案

### 3.1 現在の問題

現在のPostCardは以下の構造で、Xのツイートカードとほぼ同一：

```
[3pxカラーバー] [アバター] [名前 @handle · 3分前  ...]
                           [# テーマ名]
                           [投稿本文テキスト]
                           [属性バッジ群]
                           [返信 リポスト Good Bad 分析]
```

### 3.2 新デザイン: 「オピニオンカード」

コンセプト: **「誰が言ったか」ではなく「どの立場から言ったか」を最初に目に入れる**

#### 3.2.1 レイアウト構造

```
[トピックラベル（あれば）]
+-------------------------------------------------------+
|                                                       |
|  投稿本文テキスト（メインコンテンツ）                      |
|  最大4行表示。それ以上は「続きを読む」                     |
|                                                       |
+-------------------------------------------------------+
|  [バックグラウンドタグ群（横スクロール）]                   |
|  [30代] [会社員] [やや左派]                              |
+-------------------------------------------------------+
|  [アバター(小)] 表示名 · 3分前                           |
+-------------------------------------------------------+
|  共感 12    異論 3    返信 5    リポスト 2      [...]    |
+-------------------------------------------------------+
```

#### 3.2.2 デザインの意図

1. **本文が最上部**: 「何を言ったか」が最初に目に入る。Xは「誰が言ったか」（名前）が先頭だが、My Opinionは意見そのものを優先する。

2. **バックグラウンドタグが本文の直後**: 意見を読んだ直後に「どの立場の人の意見か」がわかる。意見の解釈に不可欠な情報を、本文に寄り添わせる。

3. **著者情報は控えめに下部**: 名前・ハンドルは小さく下部に。「個人」ではなく「意見」にフォーカスする思想を体現。

4. **アクションバーは最下部**: 機能は維持するが、意見とバックグラウンドを読んだ後に評価する導線。

5. **カラーバーを左端ではなく上部テーマラベルの色として使用**: テーマに紐づく投稿は、テーマ名ラベルの背景色としてアクセントを付ける。テーマなしの投稿はラベルなし。

#### 3.2.3 Tailwind実装仕様

```tsx
// PostCard 新レイアウト
<article className="border-b bg-card transition-all duration-200 hover:bg-accent/30">
  <div className="px-4 pt-4 pb-3">

    {/* トピックラベル（テーマ紐づき時のみ） */}
    {themeName && (
      <div className="mb-2">
        <span className="inline-flex items-center gap-1 rounded-full bg-primary/8 px-2.5 py-0.5 text-xs font-medium text-primary">
          {themeName}
        </span>
      </div>
    )}

    {/* 本文（メイン） */}
    <div onClick={navigateToPost} className="cursor-pointer">
      <p className="text-[15px] leading-relaxed whitespace-pre-wrap break-words">
        {content}
      </p>
    </div>

    {/* バックグラウンドタグ群 */}
    {attributes.length > 0 && (
      <div className="mt-3 flex flex-wrap gap-1.5">
        {attributes.map(attr => (
          <AttributeBadge key={attr.type} type={attr.type} value={attr.value} size="md" />
        ))}
      </div>
    )}

    {/* 著者行（控えめ） */}
    <div className="mt-3 flex items-center gap-2">
      <UserAvatar src={avatarUrl} displayName={displayName} size="xs" />
      <span className="text-xs text-muted-foreground">
        {displayName} · {relativeTime}
      </span>
      <div className="ml-auto">
        {/* ... メニューボタン */}
      </div>
    </div>

    {/* アクションバー */}
    <div className="mt-2 flex items-center gap-4 -ml-2">
      {/* 共感ボタン */}
      <Button variant="ghost" size="sm" ...>
        <ThumbsUp /> {goodCount > 0 && goodCount}
      </Button>
      {/* 異論ボタン */}
      <Button variant="ghost" size="sm" ...>
        <ThumbsDown />
      </Button>
      {/* 返信ボタン */}
      <Button variant="ghost" size="sm" ...>
        <MessageCircle /> {replyCount > 0 && replyCount}
      </Button>
      {/* リポストボタン */}
      <Button variant="ghost" size="sm" ...>
        <Repeat2 /> {repostCount > 0 && repostCount}
      </Button>
      {/* 分析（本人のみ） */}
      {isOwnPost && (
        <Button variant="ghost" size="sm" className="ml-auto" ...>
          <BarChart3 /> レスポンス分析
        </Button>
      )}
    </div>
  </div>
</article>
```

#### 3.2.4 UserAvatarに `xs` サイズの追加

著者行で使用するため、`UserAvatar`に `xs` サイズ（24x24px）を追加する。

```typescript
// UserAvatar.tsx に追加
const sizeMap = {
  xs: "h-6 w-6 text-[10px]",   // 新規
  sm: "h-8 w-8 text-xs",
  md: "h-10 w-10 text-sm",
  lg: "h-16 w-16 text-lg",
};
```

#### 3.2.5 左端カラーバーの廃止

現在のPostCardにある左端3pxの `barColor` カラーバーは廃止する。理由：

- Xのデザインに似た縦線があると差別化にならない
- テーマラベルの色でアクセントを担う
- バックグラウンドタグの色分け（Rose系/Emerald系/Indigo系等）が十分に立場を可視化している

#### 3.2.6 フィードの見せ方（変更なし）

フィード全体の構造（縦スクロールの投稿一覧 + InfiniteScroll）はそのまま維持する。理由：

- グリッド型やマガジン型に変えると、投稿の本文量に対してレイアウトが崩れやすい
- テーマ詳細には既に「意見マップ（スペクトラム）」タブがあり、フィードとは異なる可視化が共存している
- ホーム画面は既にテーマカード + 横スクロール + 投稿一覧の混合レイアウトでXと差別化済み

投稿カード自体のレイアウトを変えることで、同じ縦スクロールでも「Xのフィード」とは明確に異なる印象になる。

## 5. 実装ガイド（フロントエンドエンジニア向け）

### 5.1 実装順序（推奨）

1. **PostCard レイアウト変更**（最大のビジュアル変更）
   - 構造を「本文→バックグラウンドタグ→著者→アクション」に並び替え
   - 左端カラーバー削除
   - UserAvatarに `xs` サイズ追加
   - テーマラベルをカラーチップに変更

2. **Good/Bad → 共感/異論 の全画面置換**
   - PostCard、分析ページ、スペクトラム、設定画面
   - UIテキストのみ。API・DBフィールド名は変更しない

3. **ナビゲーション用語変更**
   - BottomNav、Sidebar、Header

4. **主要画面の用語変更**
   - ホーム画面（セクション見出し）
   - テーマ詳細（タブ名、ボタンテキスト）
   - 探索画面（タブ名、空状態テキスト）

5. **コンポーネント内の用語変更**
   - AttributeLensBar（「属性レンズ」→「視点フィルター」）
   - ConsensusMeter（「コンセンサス」→「意見の一致度」）
   - OpinionSpectrum（「オピニオン・スペクトラム」→「みんなの意見マップ」）
   - PositionMap（「マイ・ポジションマップ」→「わたしの立ち位置」）

6. **設定・プロフィール編集の用語変更**
   - 設定画面のリンクラベル、通知設定ラベル
   - プロフィール編集の「属性」→「プロフィール」

7. **オンボーディングの用語変更**
   - Step2, Step3, Step4のテキスト

8. **ComposeModal / ReportModal の用語変更**

### 5.2 注意事項

1. **コンポーネント名・ファイル名は変更しない**: `AttributeBadge`, `AttributeLensBar`, `OpinionSpectrum`, `ConsensusMeter`, `PositionMap` 等はそのまま。UIに表示される文字列のみ変更。

2. **API・DBフィールド名は変更しない**: `good` / `bad`, `theme`, `attribute` 等のAPIキーは維持。フロントエンドの表示レイヤーでのみ変換。

3. **URLパスは変更しない**: `/themes/:id`, `/settings/echo-chamber` 等はそのまま。

4. **PostCard props の変更**: PostCardのpropsに `themeName` が既にあるので、新レイアウトでもそのまま利用可能。構造の並び替えとスタイル変更が主な作業。

5. **Good/Bad → 共感/異論**: 投稿分析ページの Recharts の `dataKey` は `"Good"` / `"Bad"` のまま維持し、凡例の `name` prop で表示名を変更する。
