# My Opinion UX抜本改修 v4

**作成日**: 2026-02-09  
**目的**: X類似の消費型フィード体験から離れ、My Opinion固有の「意見可視化SNS」体験を中心に再構築する

---

## 1. 問題整理

現状はトピック機能や可視化機能を持つ一方、操作の主導線が依然として投稿閲覧中心で、ユーザー体験が「Xに似たSNS」に収束しやすい。

主な課題:
- トピックがあるのに、入り口が「タイムライン閲覧」に見えやすい
- 意見の分布や対立構造が、タブ奥の機能として埋もれている
- ナビゲーションが機能一覧的で、体験ストーリーを誘導していない

---

## 2. 新コンセプト

### 2.1 体験メタファー

**Opinion Observatory（意見観測所）**

ユーザーは投稿を読むだけでなく、以下を行う:
1. いま社会で温度差が大きいトピックを見つける
2. 立場別の声を対比して構造を理解する
3. 自分の意見を投じて、地図上の位置を確認する

### 2.2 コアUXループ

1. **発見**: ホームで「分断中/収束中/合意形成」を俯瞰
2. **理解**: テーマ詳細で「対比ビュー」と「意見マップ」を往復
3. **参加**: 意見投稿で議論に参加し、立ち位置変化を追う

---

## 3. 情報設計の変更

### 3.1 ホーム (`/home`)

- 役割を「タイムライン」ではなく「観測ハブ」に固定
- セクション再編:
  - 観測中メイントピック
  - 温度差で見るトピック（分断中 / 収束中 / 合意形成）
  - 最近の声（補助導線）

### 3.2 テーマ詳細 (`/themes/:id`)

- 役割を「議論アリーナ」に固定
- タブ構成:
  - みんなの声
  - 対比ビュー（新規）
  - 意見マップ
- 対比ビューで、共感寄り/中立寄り/異論寄りの声を同時表示

### 3.3 ナビゲーション

- `Sidebar` と `BottomNav` をトピック起点に統一
- 右カラムは投稿フォームだけでなく「観測カード」を置く

---

## 4. UI原則

1. **比較可能性を優先**: 一覧より対比を強調
2. **データを主役に**: 一致度・分布・偏りを常時可視化
3. **文脈付き投稿体験**: 意見はトピック文脈に紐づけて表示
4. **X類似の回避**: 単純時系列フィード中心の見え方を避ける
