# My Opinion Product Concept v5: X Intelligence Layer

**作成日**: 2026-02-13  
**目的**: コールドスタート障壁を超えるため、Xデータを一次情報源として活用しつつ、独自スタンス分析価値を提供する

---

## 1. なぜ v5 が必要か

既存の「ユーザー投稿起点SNS」は、初期参加者が少ないと可視化価値が発生しにくい。
可視化に十分なサンプルを貯める前に離脱が起き、収益化まで到達しづらい。

v5 では次の方針に切り替える:

1. 論点データの一次在庫は X API から取得する
2. My Opinion は「投稿場」ではなく「論点構造の分析・可視化レイヤー」に集中する
3. 独自投稿は維持するが、価値の起点を外部データ依存へ切り替える

---

## 2. 新しい価値提案

### 2.1 一言でいうと

**「Xで発生した議論を、立場構造として読めるメディア」**

### 2.2 ユーザー価値

- いまXで伸びている論点を、分断度・一致度で俯瞰できる
- 賛否の温度差を時系列で追える
- 自分のスタンスを入力すると、論点ごとの変化（ドリフト）を確認できる

### 2.3 収益価値

- B2Cメディアとして閲覧流入を確保しやすい
- プレミアム会員で「高度比較・履歴分析・保存」を課金化できる

---

## 3. 実装された最小基盤（今回）

### 3.1 データ基盤

新規テーブル（`00004_add_x_intelligence_tables.sql`）:

- `x_issues`: 外部由来の論点
- `x_issue_posts`: 論点紐づきの外部ポストキャッシュ
- `user_issue_stances`: ユーザーの最新スタンス
- `user_issue_stance_events`: スタンス履歴（ドリフト分析用）

### 3.2 API

- `GET /api/x/issues/trending`
- `GET /api/x/issues/:issueId/posts`
- `GET /api/x/issues/:issueId/insights`
- `GET /api/x/issues/:issueId/stance`
- `POST /api/x/issues/:issueId/stance`
- `GET /api/me/stance-drift`

### 3.3 ポリシー

- 外部本文は `X_TEXT_POLICY` で表示ポリシーを切替
- データ由来情報（`DataProvenance`）を保持
- 推定属性は補助情報としてのみ扱い、断定しない
