**作成者**: プロダクトデザイナー

## 1. Xとの類似点・問題点の分析

### 1.1 構造的な類似点

現在のMy Opinionは、UI/UXの面でX (Twitter) とほぼ同一の構造を採用しており、「属性開示型SNS」というユニークな価値が埋もれてしまっている。

| 要素 | X (Twitter) | 現在のMy Opinion | 問題点 |
|------|-------------|-----------------|--------|
| **レイアウト** | 3カラム（サイドバー / メインフィード / 右パネル） | 同一の3カラム構成。コード内コメントに「like X/Twitter」と明記 | Xの第一印象をそのまま再現しており、初見で差別化が伝わらない |
| **タイムライン** | 縦スクロールの投稿カード一覧 | 同一。PostCardコンポーネントがXのツイートカードそのもの | 「属性」が投稿の付随情報に過ぎず、コア体験として機能していない |
| **投稿カード** | アバター / 表示名 / @ハンドル / 時間 / 本文 / アクションバー | 同一構成。Good/Badボタンの位置もいいね/リツイートと同じ | 属性バッジは小さく目立たない。投稿の「誰が言ったか」より「何を言ったか」に視線が行く |
| **ナビゲーション** | ホーム / 探索 / 通知 / メッセージ / プロフィール | ホーム / 探索 / テーマ / 通知 / プロフィール | テーマがメッセージの置き換えに過ぎない印象 |
| **プロフィール** | バナー / アバター / 名前 / bio / フォロー数 / 投稿一覧 | 同一構成（グラデーションバナー + アバター + 投稿一覧） | 属性が小さなバッジとして並ぶだけで、Xのプロフィールと区別がつかない |
| **投稿作成** | テキスト入力 + 送信ボタン | 同一。ComposeModal / ComposePanel | テーマ選択が「任意」のため、ほぼ自由投稿SNSになりがち |
| **ボトムナビ** | 5つのアイコン横並び | 同一パターン | モバイルでのXとの見分けが不可能 |

### 1.2 コアバリューが活かされていない点

1. **属性バッジが「おまけ」に見える**: PostCard内で属性バッジは投稿者名の下に小さく表示されるだけ。Xの「認証バッジ」と同程度の存在感しかなく、「属性開示型」というコアコンセプトが体験に組み込まれていない。

2. **エコーチェンバー指標がサイドバーの隅に追いやられている**: Sidebarにミニウィジェットとして存在するが、/settings/echo-chamber という深い階層に導線があるだけ。プロダクト最大の差別化要素なのに、ユーザーの日常動線に組み込まれていない。

3. **分析機能が「投稿者本人のみ」で閉じている**: 投稿分析は価値が高いが、投稿者本人しか見られないため、プラットフォーム全体としての「属性別の意見の分かれ方」が可視化されていない。

4. **テーマが単なるカテゴリラベル**: テーマは投稿を分類するタグのように機能しているだけで、「属性開示 x 議論」のダイナミズムを活かしていない。

5. **タイムラインがXの時系列フィードそのもの**: 「フォロー中の人の投稿を時系列で見る」というX的体験がそのまま踏襲されており、「多様な意見に出会う」というミッションと矛盾している。

### 1.3 根本的な問題

**My OpinionはXの「スキン違い」になっている。** ユーザーは「もう一つのX」としか認識せず、わざわざ別プラットフォームを使う動機がない。属性開示・エコーチェンバー可視化・テーマ制議論という3つのコアバリューが、Xのフレームワークに無理やり詰め込まれた結果、どれも中途半端な存在感になっている。

## 2. 新しいプロダクトコンセプト

### 2.1 ビジョンステートメント

> **「意見の地図を描く」** --- My Opinionは、社会の意見を属性ごとに可視化する「オピニオン・マッピング」プラットフォーム。タイムラインをスクロールする場所ではなく、**社会全体の意見の分布を探索し、自分の立ち位置を発見する場所**。

### 2.2 コンセプトの転換

| | 現在（X的モデル） | 提案（オピニオン・マッピング） |
|---|---|---|
| **中心体験** | 個人の投稿をフォローして読む | テーマごとの意見の分布を探索する |
| **投稿の意味** | 自分のフォロワーに向けた発信 | テーマに対する「一票」であり社会的な声 |
| **属性の役割** | プロフィールの付随情報 | 意見を地図上にプロットするための座標軸 |
| **フィードの設計** | 時系列スクロール | テーマ中心 + 属性フィルター + 意見スペクトラム表示 |
| **評価の意味** | 個人への共感/否定 | 意見への同意/異論のデータポイント |
| **分析の公開性** | 投稿者本人のみ | テーマ全体の分析は全員に公開（個人投稿分析は本人のみ） |

### 2.3 メタファーの変更

- **旧**: My OpinionはSNS（ソーシャル・ネットワーキング・サービス）
- **新**: My Opinionは**「社会の意見のリアルタイム世論調査 + 議論プラットフォーム」**

XやFacebookのような「人をフォローしてフィードを読む」モデルから、**「テーマを選んで、社会の意見の分布を見て、自分の意見を投じる」**モデルへの転換。

## 3. 具体的な新機能・新体験の提案

### 提案1: オピニオン・スペクトラム（Opinion Spectrum）

**概要**: テーマに対する全投稿を、属性軸で並べた「意見の分布図」として表示する新しいメインビュー。タイムラインの代替。

**詳細**:
- テーマを開くと、まず「意見スペクトラム」が表示される
- 横軸に属性（例: 政治スタンス「左派 -- 中道 -- 右派」）、縦軸にGood/Bad比率を取った散布図 or ヒートマップ
- 各ドットが1つの投稿を表し、タップ/クリックで投稿全文と投稿者属性を表示
- 上部で軸にする属性を切り替え可能（性別、年齢帯、職業、政治スタンスなど）
- 自分の投稿がスペクトラム上のどこに位置するかがハイライトされる

**なぜ斬新か**: 既存のどのSNSにも「意見を属性ごとにマッピングして俯瞰する」機能は存在しない。NewsPicksはコメントを並べるだけ、Xはタイムラインを流すだけ。My Opinionだけが属性データを持っているからこそ可能な体験。

**ユーザーインパクト**: 高。「自分の意見が社会のどの位置にあるか」を直感的に理解でき、エコーチェンバーの自覚を最も強力に促す。
**実装難易度**: 中。Rechartsの散布図/ヒートマップで実装可能。データはreactions + user_attributesのJOINで取得。
**優先度**: 最高（コア体験の刷新）

---

### 提案2: テーマ・ファースト・ナビゲーション

**概要**: ナビゲーションの中心を「テーマ」に置き換える。ホーム画面にアクセスすると、まず「今週のテーマ」「注目のテーマ」が表示される。

**詳細**:
- ホーム画面の構成を根本的に変更:
  - **上段**: 今週の運営テーマ（大きなカード形式。テーマタイトル + 参加者数 + 投稿数 + 「意見を投じる」ボタン）
  - **中段**: 注目のテーマ一覧（横スクロールカード。Good/Bad比率のミニグラフ付き）
  - **下段**: フォロー中の人の最新活動（従来のタイムラインの簡略版）
- ボトムナビの「ホーム」アイコンを「テーマ一覧」に変更
- 「探索」は属性フィルター付きの全テーマ検索に進化

**なぜ斬新か**: 既存SNSは全て「人→投稿」の構造。My Opinionは「テーマ→意見→属性」の構造にすることで、「議論に参加するプラットフォーム」という独自のポジションが明確になる。

**ユーザーインパクト**: 高。サービスの第一印象が「Xの代替品」から「社会の議論に参加する場」に変わる。
**実装難易度**: 低。既存のテーマAPIとタイムラインAPIの組み合わせで実装可能。UIコンポーネントの再構成が主。
**優先度**: 最高（ファーストインプレッションの刷新）

---

### 提案3: 属性レンズ（Attribute Lens）

**概要**: 任意の投稿一覧やテーマタイムラインに対して、「属性レンズ」をかけることで、特定属性のユーザーの投稿だけをフィルターして表示する機能。

**詳細**:
- テーマタイムラインの上部に「属性レンズバー」を常時表示
- 「20代の意見」「女性の意見」「左派の意見」などの属性条件を選択可能
- 複数属性の掛け合わせも可能（「30代 x 会社員」など）
- レンズを切り替えるたびに、フィード内容がアニメーションで切り替わる
- 現在の「探索ページ」のフィルターチップを進化させたもの

**なぜ斬新か**: 探索ページに既にフィルター機能はあるが、「レンズを切り替えることで意見の多様性を実感する」という能動的な体験に昇華させる。単なるフィルターではなく、「別の立場から社会を見る」メタファーを持たせる。

**ユーザーインパクト**: 中〜高。「エコーチェンバーから抜け出す」体験を能動的に提供。
**実装難易度**: 低。既存の探索ページのフィルター機能の拡張。UIの改善が主。
**優先度**: 高

---

### 提案4: マイ・ポジション・マップ（My Position Map）

**概要**: ユーザーのプロフィールページに「自分の意見マップ」を表示。過去の投稿・評価の傾向から、属性軸上での自分の立ち位置を可視化する。

**詳細**:
- 現在のエコーチェンバー指標を大幅に拡張
- レーダーチャートで「自分の意見の偏り」を6軸（テーマごとの賛否傾向）で可視化
- 「あなたと意見が近いユーザー」「あなたと意見が遠いユーザー」のレコメンド
- 時系列での自分の立ち位置の変化を表示（月次推移）
- プロフィールページの最上部に大きく配置（投稿一覧より上位に表示）

**なぜ斬新か**: 既存SNSには「自分の投稿・いいねの傾向を分析して見せる」機能が全くない。Spotifyの年間まとめ（Wrapped）のような「自分を知る」体験をSNSの日常機能として提供する。

**ユーザーインパクト**: 高。自己理解・自己発見の体験は強力なエンゲージメントドライバー。
**実装難易度**: 中。Rechartsのレーダーチャート + 既存のecho-chamber APIの拡張。
**優先度**: 高

---

### 提案5: コンセンサス・メーター（Consensus Meter）

**概要**: テーマ全体の「社会的合意度」をリアルタイムに可視化するインジケーター。テーマに紐づく全投稿のGood/Bad評価の分布から、「この議題はどれくらい意見が割れているか」を表示。

**詳細**:
- テーマ詳細ページのヘッダーに大きく表示
- 0%（完全に意見が割れている）〜 100%（全員が同意見）のスケール
- 属性別のコンセンサス度合い（「年齢帯で最も意見が割れている」「政治スタンスで最も意見が一致している」等）
- テーマに投稿するたびにメーターがリアルタイムに動く（ゲーミフィケーション要素）
- テーマカードにミニコンセンサスメーターを表示し、「意見が割れているテーマ」に自然と興味を引かれる導線

**なぜ斬新か**: 世論調査は結果が出るまで見えない。My Opinionなら投稿ベースでリアルタイムに「社会の意見の分かれ具合」を可視化できる。テレビ番組の生投票のような即時性と、学術的な世論調査の深さを両立。

**ユーザーインパクト**: 中〜高。テーマへの参加モチベーション向上 + 「社会を可視化している」感覚の強化。
**実装難易度**: 中。既存のreactions + theme_postsデータの集計。計算ロジックの設計がポイント。
**優先度**: 中〜高

---

### 提案6: 対比ビュー（Contrast View）

**概要**: 同一テーマに対する2つの属性グループの投稿を左右に並べて表示する「対比ビュー」。

**詳細**:
- テーマ詳細ページに「対比モード」の切り替えボタンを追加
- 属性軸を1つ選択すると、画面が左右に分割される
  - 例: 「左派の意見」と「右派の意見」
  - 例: 「20代の意見」と「50代の意見」
  - 例: 「会社員の意見」と「学生の意見」
- 左右の投稿は独立してスクロール可能
- 中央に「共通点」「相違点」のサマリーを表示（将来的にAI要約を追加可能）

**なぜ斬新か**: 既存プラットフォームは意見を混ぜて表示するか（X）、そもそも分けない（Facebook）。意見を明示的に「対比」して見せることで、エコーチェンバーの構造を体験的に理解させる。

**ユーザーインパクト**: 高。「こんなに意見が違うのか」という発見がプロダクトの核心的体験。
**実装難易度**: 中。属性フィルター付きの投稿取得APIは既に存在（explore API）。UIの左右分割が主な実装。
**優先度**: 中

---

### 提案7: ウィークリー・オピニオンレポート

**概要**: 毎週のテーマ終了時に、そのテーマに関する「意見レポート」を自動生成して全ユーザーに配信。

**詳細**:
- 週次テーマ終了後、以下を含むレポートを自動生成:
  - テーマの総投稿数・参加者数
  - 全体のGood/Bad比率
  - 属性別の意見傾向の可視化
  - 最も議論を呼んだ投稿（Good + Bad数が最多）
  - コンセンサス度合い
  - 自分の投稿の位置づけ（参加者のみ）
- 通知として配信 + 専用のレポートページで閲覧可能
- レポートのシェアカード生成（SNS拡散用）

**なぜ斬新か**: 既存SNSには「まとめ」がない。議論の成果を可視化することで、「My Opinionで発言する意味がある」と感じさせる。ジャーナリズム的価値も生む。

**ユーザーインパクト**: 中。定期的なリテンション施策 + 外部拡散の強力なフック。
**実装難易度**: 中〜高。集計ロジック + レポートUIの新規作成。自動化バッチ処理が必要。
**優先度**: 中（Phase 2に適切だがMVPでも簡易版は有用）

## 5. MVP推奨範囲

### 5.1 今回のリデザインで実装する範囲

今回のMVPリデザインでは、**提案1〜5を実装範囲**として推奨する。これにより、My OpinionはXの「スキン違い」から完全に脱却し、「属性開示 x 意見の可視化」というユニークなポジションを確立できる。

#### 優先順位と実装順序

1. **テーマ・ファースト・ナビゲーション**（提案2）
   - 実装難易度が最も低く、サービスの第一印象を劇的に変える
   - ホーム画面をテーマ中心に再構成
   - ナビゲーション構造の見直し

2. **属性レンズ**（提案3）
   - 既存の探索ページのフィルター機能を拡張・昇華
   - テーマタイムラインにも属性レンズを統合
   - UIの改善が主であり、バックエンドの変更は最小限

3. **オピニオン・スペクトラム**（提案1）
   - テーマ詳細ページに新しいビューとして追加
   - 散布図/ヒートマップの実装
   - 投稿データと属性データのクロス集計API

4. **コンセンサス・メーター**（提案5）
   - テーマカードとテーマ詳細ページに追加
   - 集計APIの実装
   - UIコンポーネントの新規作成

5. **マイ・ポジション・マップ**（提案4）
   - プロフィールページの再設計
   - エコーチェンバー指標の拡張・メイン画面への昇格
   - レーダーチャートの実装

### 5.2 既存機能との関係

- **残すもの**: 投稿・返信・リポスト・Good/Bad評価・フォロー・通知・招待コード・通報・管理者機能・投稿分析（本人限定）・シェアカード
- **変更するもの**: ホーム画面（テーマファーストに）、ナビゲーション構成、テーマ詳細ページ（スペクトラム追加）、プロフィールページ（ポジションマップ追加）、探索ページ（属性レンズ強化）
- **廃止するもの**: なし（既存機能は全て維持、表示の優先度が変わるのみ）

### 5.3 期待する効果

1. **差別化の確立**: 初めてサービスを訪れたユーザーが「これはXとは違う」と5秒で理解できる
2. **コアバリューの体験化**: 属性開示とエコーチェンバー可視化が「おまけ」ではなく「メイン体験」になる
3. **エンゲージメント向上**: テーマ参加・意見投稿の動機が「自分の投稿を見てもらう」から「社会の意見マップに参加する」に変わる
4. **拡散力の強化**: オピニオン・スペクトラムやコンセンサス・メーターのスクリーンショットが「これ何？」と興味を引く独自ビジュアルになる

## 6. 技術的な実現可能性の補足

全ての提案は現在の技術スタック（Next.js + Supabase + Recharts + shadcn/ui + Tailwind CSS）で実現可能。

| 提案 | 主な技術的要素 |
|------|--------------|
| オピニオン・スペクトラム | Rechartsの ScatterChart + reactions JOIN user_attributes のAPI |
| テーマ・ファースト・ナビ | 既存コンポーネントの再構成。新規APIは不要 |
| 属性レンズ | 探索ページの既存フィルターUI拡張。APIは既に属性フィルター対応済み |
| マイ・ポジション・マップ | RechartsのRadarChart + echo-chamber API の拡張 |
| コンセンサス・メーター | theme_posts + reactions の集計SQL。フロントはプログレスバーUI |

データベーススキーマの変更は不要。既存のreactions.reactor_attribute_snapshot（JSONB）に全ての属性情報が保存されており、クロス集計に必要なデータは既に揃っている。
