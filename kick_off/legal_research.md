## 1. 個人情報保護法関連

**リスクレベル: 高**
**対応優先度: MVP前（必須）**

### 1.1 政治的見解・信条は「要配慮個人情報」に該当するか

**結論: 該当する。**

個人情報保護法第2条第3項に定める「要配慮個人情報」には、本人の**人種、信条、社会的身分、病歴、犯罪の経歴、犯罪により害を被った事実**等が含まれる。「信条」には**政治的な主義・思想・支持政党**が含まれ、本サービスで扱う「支持政党・政治スタンス」は要配慮個人情報に該当する。

#### 取得要件

| 要件 | 内容 |
|------|------|
| **本人の事前同意（オプトイン）** | 要配慮個人情報の取得には、**あらかじめ本人の明示的な同意**が必須。黙示の同意やオプトアウト方式は不可 |
| **第三者提供の制限** | 要配慮個人情報はオプトアウト方式による第三者提供が**禁止**されている。提供には本人の個別同意が必要 |
| **利用目的の特定** | 「SNS上での属性表示」「評価分析への利用」等、具体的な利用目的を明示 |

#### 対策

- ユーザー登録時に、政治スタンス入力画面で**個別の同意取得フロー**を実装する
- 同意画面では「要配慮個人情報であること」「公開される範囲」「利用目的」を明示する
- 同意の記録（日時・内容・バージョン）を保存する仕組みを構築する
- 同意撤回の仕組みを用意し、撤回時にはデータ削除または非公開化する

### 1.2 学歴・職業・年収帯の取り扱い

**結論: 学歴・職業・年収帯は要配慮個人情報には該当しない。ただし「個人情報」に該当する。**

学歴・職業・年収帯は個人情報保護法上の要配慮個人情報の11カテゴリ（人種、信条、社会的身分、病歴、犯罪の経歴、犯罪被害の事実、身体障害・知的障害・精神障害等、健診結果、医師等の指導・診療・調剤、刑事手続き、少年保護事件手続き）に含まれないが、他の情報と組み合わせて個人を特定できる場合は個人情報に該当する。

#### 取得要件

| 要件 | 内容 |
|------|------|
| **利用目的の通知・公表** | 取得時に利用目的を本人に通知するか、あらかじめ公表する |
| **本人同意** | 第三者提供する場合は本人の事前同意が必要（SNS上での表示は本人の自発的公開と整理可能） |
| **適正取得** | 偽りその他不正の手段による取得の禁止 |

#### 対策

- プロフィール設定画面で、各属性の公開範囲を選択できる機能を実装
- 利用目的を明確にプライバシーポリシーに記載
- 年収帯は範囲表示（例：400-600万円）に限定し、正確な金額は収集しない

### 1.3 プライバシーポリシーに必要な記載事項

以下の項目をプライバシーポリシーに記載する必要がある:

1. **事業者の名称・住所・代表者名**
2. **個人情報の定義**（本サービスにおける個人情報の範囲）
3. **取得する個人情報の項目**（属性情報、投稿内容、評価データ等を具体的に列挙）
4. **利用目的**（属性表示、評価分析、サービス改善、統計情報の作成等を具体的に）
5. **要配慮個人情報の取り扱い**（政治スタンス等が該当する旨と同意取得方法）
6. **第三者提供の有無と条件**（提供先の範囲・属性を示す）
7. **共同利用の有無**
8. **委託先への提供**
9. **本人からの開示・訂正・削除・利用停止請求の対応方法**
10. **Cookie・アクセス解析ツールの利用**（外部送信規律への対応を含む）
11. **データの保存期間・削除方針**
12. **安全管理措置の概要**
13. **問い合わせ窓口**
14. **プライバシーポリシーの変更手続き**

### 1.4 個人情報保護委員会への届出

**結論: 個人情報取扱事業者としての届出義務は、現行法では原則不要。**

2017年の法改正により、個人情報データベース等を事業の用に供している者は自動的に「個人情報取扱事業者」となり、以前あった主務大臣への届出制度は廃止された。ただし、個人情報保護委員会からの報告徴収・立入検査には対応義務がある。

**注意**: 2026年以降の法改正で届出制度が復活する可能性もあるため、改正動向を継続的にモニタリングすること。

### 1.5 属性 x 評価の分析データの取り扱い

**リスクレベル: 高**

属性情報と評価データを組み合わせた分析データは、以下の観点で慎重な取り扱いが必要:

| リスク | 内容 | 対策 |
|--------|------|------|
| **プロファイリング** | 属性 x 評価の分析により、特定個人の思想傾向が推定される | 分析結果は集計・統計データに限定し、個人レベルの分析結果を第三者に提供しない |
| **差別助長** | 「XX属性の人はYY傾向」等の分析結果が差別に利用される | 表示する統計は一定数以上のサンプルに限定（例: 10人以上） |
| **要配慮個人情報の推知** | 投稿内容やGood/Bad評価パターンから政治的傾向が推知される可能性 | 個人の評価履歴は本人のみ閲覧可とする |
| **匿名加工情報** | 統計データとして外部提供する場合は匿名加工情報として適切に加工 | 個人情報保護委員会規則に従った加工基準を適用 |

## 2. プロバイダ責任制限法（情報流通プラットフォーム対処法）

**リスクレベル: 高**
**対応優先度: MVP前（必須）**

### 2.1 法律の現状

2024年5月に改正法が成立し、2025年4月1日に施行。法律名も「特定電気通信による情報の流通によって発生する権利侵害等への対処に関する法律」（通称: 情報流通プラットフォーム対処法）に改称された。

### 2.2 SNS運営者としての責任範囲

本サービスは「特定電気通信役務提供者」に該当する。運営者の責任範囲は以下の通り:

| 項目 | 内容 |
|------|------|
| **損害賠償責任の制限** | 権利侵害情報を知らなかった場合、被害者への賠償責任は制限される |
| **削除の免責** | 正当な理由に基づき投稿を削除した場合、発信者への賠償責任は制限される |
| **大規模事業者の義務** | 月間発信者数1,000万人以上等の場合、追加の義務が発生（MVP段階では非該当） |

### 2.3 発信者情報開示請求への対応義務

- 被害者からの開示請求に対し、**発信者情報（IPアドレス、メールアドレス等）の開示義務**がある
- 2022年改正で導入された「発信者情報開示命令」手続きにより、1回の裁判手続きで開示可能
- **対策**: 発信者情報開示請求への対応フローを整備し、法務担当（または顧問弁護士）への連絡体制を構築する

### 2.4 投稿の削除基準・対応フロー

以下の削除基準を策定し、公表する必要がある:

**削除対象とすべき投稿カテゴリ:**

1. **権利侵害情報**: 名誉毀損、プライバシー侵害、著作権侵害、肖像権侵害
2. **差別的表現**: 属性に基づく差別的言動、ヘイトスピーチ
3. **違法情報**: 犯罪行為の教唆・幇助、薬物・武器等の取引情報
4. **規約違反**: 虚偽の属性情報、なりすまし行為

**対応フロー:**

1. ユーザーまたは第三者からの通報受付（24時間受付窓口）
2. 通報内容の確認・一次判断（1営業日以内）
3. 削除判断（権利侵害の明白性を評価）
4. 削除実施または非削除の決定と理由通知
5. 不服申立てプロセス
6. 記録の保存（通報から対応完了まで）

## 3. 名誉毀損・誹謗中傷対策

**リスクレベル: 高**
**対応優先度: MVP前（必須）**

### 3.1 属性情報を利用した差別的投稿への法的リスク

本サービスは属性を公開して議論するという特性上、属性に基づく差別的投稿が発生するリスクが**通常のSNSより高い**。

| リスク | 法的根拠 | 内容 |
|--------|----------|------|
| **名誉毀損** | 刑法230条 | 「XX大学卒のくせに」等、属性を根拠とした人格攻撃 |
| **侮辱** | 刑法231条 | 2022年厳罰化（1年以下の懲役・禁錮または30万円以下の罰金） |
| **不法行為** | 民法709条 | 損害賠償請求の対象 |
| **運営者の共同不法行為** | 民法719条 | 違法投稿を放置した場合、運営者も責任を問われる可能性 |

### 3.2 ヘイトスピーチ解消法との関連

**リスクレベル: 中**

「本邦外出身者に対する不当な差別的言動の解消に向けた取組の推進に関する法律」（ヘイトスピーチ解消法、2016年施行）は**理念法であり罰則なし**だが、以下の点で本サービスに影響する:

- 地方自治体の条例（大阪市、川崎市等）では罰則付きの規制あり
- 法務省の人権擁護機関がSNS上のヘイトスピーチを監視・削除要請を行っている
- 本サービスでは「政治スタンス」等が公開されるため、特定の政治的立場への集団攻撃が起きやすい環境
- **対策**: ヘイトスピーチだけでなく、**属性全般に基づく差別的言動**を禁止する独自のポリシーを策定

### 3.3 運営者が取るべきモデレーション方針

| 施策 | 内容 | 優先度 |
|------|------|--------|
| **コミュニティガイドライン** | 禁止行為を具体的に列挙（属性差別、人格攻撃、なりすまし等） | MVP前 |
| **通報機能** | ユーザーが問題投稿を通報できる仕組み | MVP前 |
| **AIフィルタリング** | 差別的表現の自動検出（将来的に導入） | MVP後 |
| **段階的制裁** | 警告 → 一時停止 → アカウント凍結の段階的対応 | MVP前 |
| **モデレーターの配置** | 通報への対応・判断を行う人員の確保 | MVP前（最低1名） |
| **透明性レポート** | 削除件数・理由等の定期公表 | MVP後 |

## 4. 著作権・知的財産

**リスクレベル: 中**
**対応優先度: MVP前**

### 4.1 ユーザー投稿の著作権帰属

- ユーザーが投稿した文章・画像等の**著作権はユーザーに帰属**する（著作権法第2条第1項第2号）
- ただし、利用規約でサービス運営に必要な範囲のライセンスを取得する必要がある

### 4.2 利用規約で定めるべきライセンス条項

```
【推奨条項の骨子】
- 著作権はユーザーに留保される
- ユーザーは運営者に対し、サービスの運営・改善・宣伝に必要な範囲で、
  非独占的・無償・サブライセンス可能な利用許諾を付与する
- ユーザーは著作者人格権を行使しないことに同意する
- 運営者はユーザーの投稿を改変しない（表示形式の調整を除く）
```

**注意点:**
- 「著作権を運営者に譲渡する」条項は消費者契約法上無効と判断されるリスクあり
- 利用許諾の範囲は必要最小限に留める

### 4.3 第三者コンテンツの引用・転載ルール

| 行為 | 法的根拠 | 要件 |
|------|----------|------|
| **引用** | 著作権法32条 | 公正な慣行に合致し、引用の目的上正当な範囲内。出所明示が必要 |
| **転載** | 著作権法の原則 | 著作権者の許諾が必要。無断転載は著作権侵害 |
| **リンク共有** | -- | リンクの共有自体は著作権侵害に該当しない |

**対策:**
- 利用規約で「第三者の著作権を侵害する投稿の禁止」を明記
- DMCA的な著作権侵害申告・対応フローを整備
- 引用ガイドラインをヘルプページに掲載

## 5. 電気通信事業法

**リスクレベル: 中**
**対応優先度: MVP前**

### 5.1 SNSサービスが電気通信事業に該当するか

**結論: 該当する可能性が高い。**

改正電気通信事業法（2023年6月施行）により、SNS・電子掲示板のような**不特定多数の利用者間でのテキスト・画像等の投稿・閲覧機能**を提供する場合、電気通信事業に該当する。

本サービスは以下の機能を有するため、「第三号事業」（登録・届出不要）に該当する可能性が高い:
- ユーザー間の投稿・閲覧機能（SNS機能）
- DM機能がある場合はより明確に該当

### 5.2 届出義務の有無と手続き

| 規模 | 分類 | 届出 |
|------|------|------|
| **利用者数1,000万人未満** | 第三号事業 | 届出不要。ただし外部送信規律への対応は必要 |
| **利用者数1,000万人以上** | 総務大臣指定事業者 | 届出が必要 |

### 5.3 外部送信規律への対応（MVP前に必要）

2023年施行の改正法で導入された「外部送信規律」への対応が必要:

- Cookie、アクセス解析ツール、広告タグ等で利用者の端末から外部にデータを送信する場合
- 以下のいずれかの対応が必要:
  1. **通知・公表**: 送信先・送信情報・利用目的をプライバシーポリシー等で公表
  2. **同意取得**: Cookie同意バナー等での同意取得
  3. **オプトアウト**: 利用者が拒否できる手段の提供

**対策**: プライバシーポリシーに外部送信に関する記載を追加し、Cookie同意バナーを実装する

## 6. 特定商取引法

**リスクレベル: 中**
**対応優先度: サブスクリプション導入前**

### 6.1 サブスクリプション課金に必要な表示事項

本サービスがサブスクリプション型課金を導入する場合、「通信販売」に該当し、以下の表示義務がある:

**最終確認画面での必須表示項目:**

1. **商品等の分量**: サブスクリプションの内容・提供期間
2. **販売価格・対価**: 月額/年額料金、初回割引がある場合は通常料金も明示
3. **支払の時期・方法**: 課金日、決済手段
4. **引渡し・提供時期**: サービス開始時期
5. **申込みの期間**: 契約期間、自動更新の有無
6. **解約条件**: 解約方法、解約時の日割り計算の有無

**特定商取引法に基づく表示（事業者情報）:**

1. 販売業者名（法人名）
2. 運営統括責任者名
3. 所在地
4. 電話番号
5. メールアドレス
6. 販売価格
7. 支払方法
8. 返品・キャンセルに関するポリシー

### 6.2 クーリングオフ制度の適用

**結論: 通信販売にはクーリングオフ制度は適用されない。**

- 特定商取引法上、クーリングオフが適用されるのは訪問販売・電話勧誘販売・連鎖販売取引等
- **通信販売（ECを含むオンラインサービス）にはクーリングオフは適用されない**
- ただし、事業者が独自に返金ポリシーを定めない場合、商品到着後8日間は返品可能（法定返品権）
- デジタルコンテンツ・サービスの場合は、利用規約で返金条件を明示することが推奨される

**対策:**
- 利用規約に解約・返金ポリシーを明記
- 「サブスクリプション解約」の導線を分かりやすく設計
- 2022年改正で義務化されたクーリングオフの電磁的方法（メール等）での受付体制を整備（該当する取引形態がある場合）

## 7. 利用規約・プライバシーポリシーの策定指針

**リスクレベル: 高**
**対応優先度: MVP前（必須）**

### 7.1 利用規約の必須条項一覧

| # | 条項 | 内容 |
|---|------|------|
| 1 | **サービスの定義** | 属性開示型SNSとしての機能・目的を明示 |
| 2 | **利用資格** | 年齢制限（18歳以上推奨）、反社会的勢力の排除 |
| 3 | **アカウント登録** | 属性情報の正確性義務、虚偽登録の禁止 |
| 4 | **属性情報の取り扱い** | 公開範囲、変更・削除の方法、要配慮個人情報に関する同意 |
| 5 | **ユーザーの権利と責任** | 投稿内容に対する責任は投稿者に帰属 |
| 6 | **知的財産権** | 著作権の帰属、運営者へのライセンス付与 |
| 7 | **禁止事項** | 下記7.3参照 |
| 8 | **モデレーションポリシー** | 投稿の削除・アカウント停止の基準と手続き |
| 9 | **免責事項** | 下記7.2参照 |
| 10 | **料金・課金** | サブスクリプションの料金体系、自動更新、解約条件 |
| 11 | **退会・データ削除** | 退会方法、退会後のデータ保持期間 |
| 12 | **規約の変更** | 変更の通知方法、変更後の利用継続による同意のみなし |
| 13 | **準拠法・管轄裁判所** | 日本法、東京地方裁判所（第一審専属管轄） |
| 14 | **反社会的勢力の排除** | 反社条項 |
| 15 | **分離可能性** | 一部条項が無効でも他の条項は有効 |

### 7.2 免責事項の設計

| 免責範囲 | 内容 | 注意点 |
|----------|------|--------|
| **投稿内容** | ユーザー投稿の正確性・適法性を保証しない | 消費者契約法により「一切の責任を負わない」は無効リスクあり |
| **サービスの継続性** | サービスの中断・停止について免責 | 重大な過失がある場合は免責不可 |
| **属性情報の正確性** | 他ユーザーの属性が正確であることを保証しない | 虚偽属性の通報・対応フローで補完 |
| **第三者とのトラブル** | ユーザー間のトラブルについて免責 | 完全な免責は消費者契約法上困難 |
| **損害賠償の上限** | 損害額の上限を利用料金相当額に制限 | 故意・重過失の場合は制限不可 |

**消費者契約法への留意:**
- 事業者の損害賠償責任を全部免除する条項は**無効**（消費者契約法8条）
- 事業者の故意・重過失による損害賠償責任を一部免除する条項は**無効**
- 「軽過失の場合に限り、利用料金を上限として賠償する」程度の制限が実務的な落としどころ

### 7.3 禁止事項の設計（属性に基づく差別禁止等）

**本サービス特有の禁止事項:**

1. **属性差別の禁止**: 学歴、職業、年収、性別、政治的立場等の属性を理由とした差別的発言・攻撃
2. **虚偽属性の禁止**: 虚偽の属性情報を登録する行為
3. **属性による排除**: 特定属性のユーザーの発言を組織的に封殺する行為（集団的Bad評価の組織化等）
4. **個人特定行為の禁止**: 属性情報の組み合わせにより特定個人を推定・晒す行為
5. **属性情報の外部流出**: 他ユーザーの属性情報をサービス外に転載する行為

**一般的な禁止事項:**

6. 名誉毀損・誹謗中傷
7. わいせつ情報の投稿
8. 犯罪行為の教唆・幇助
9. 商業目的の無断利用・スパム行為
10. 不正アクセス・システムへの攻撃
11. なりすまし行為
12. マルチアカウントの不正利用
13. 未成年者への不適切なコンタクト
14. 個人情報の不正収集
15. 法令・公序良俗に反する行為

## 8. GDPR等海外法への備え

**リスクレベル: 低（現時点）→ 将来的に高**
**対応優先度: MVP後（海外展開時）**

### 8.1 GDPRの概要と本サービスへの影響

| 項目 | 内容 |
|------|------|
| **適用条件** | EU/EEA域内の個人にサービスを提供する場合、または域内の個人の行動を監視する場合 |
| **日本の十分性認定** | 2019年1月に日本はEUから十分性認定を受けており、日EU間のデータ移転は追加的な手続き不要 |
| **ただし** | 十分性認定があっても、GDPR自体の遵守と「補完的ルール」の遵守が必要 |

### 8.2 GDPRで追加的に必要な対応

| 対応事項 | 内容 | 優先度 |
|----------|------|--------|
| **データ保護影響評価（DPIA）** | 政治的見解等のセンシティブデータを扱うため、DPIAの実施が必須 | 海外展開前 |
| **データ保護責任者（DPO）の任命** | 大規模にセンシティブデータを処理する場合、DPOの任命が必要 | 海外展開前 |
| **明示的同意** | GDPR第9条により、政治的見解は「特別カテゴリーのデータ」に該当し、**明示的同意**が必要 | 海外展開前 |
| **データポータビリティ権** | ユーザーが自分のデータを構造化された形式で受け取る権利を保証 | 海外展開前 |
| **忘れられる権利** | ユーザーからのデータ削除要請への対応義務 | 海外展開前 |
| **多言語プライバシーポリシー** | 対象言語でのプライバシーポリシー提供 | 海外展開前 |

### 8.3 その他の海外法

| 地域 | 法律 | 概要 |
|------|------|------|
| **米国カリフォルニア州** | CCPA/CPRA | 消費者のプライバシー権保護。オプトアウト権、データ削除権等 |
| **中国** | 個人情報保護法（PIPL） | 中国国内のデータ処理に関する厳格な規制。データの国外移転制限 |
| **韓国** | 個人情報保護法 | 日本法と類似するが、より厳格な同意取得要件 |
