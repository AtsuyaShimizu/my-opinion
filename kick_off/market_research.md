### 1.2 属性開示型・実名制SNSの国内外事例

#### LinkedIn（実名＋職業属性開示の成功事例）
- **日本ユーザー数**: 約509万人（2025年3月）、前年比約20%成長
- **グローバル売上**: 年間$178億（FY2025）、Premium課金だけで年間$17億
- **成功要因**: 職業属性（会社名・役職・学歴）の開示がプラットフォームの核心価値。採用・ビジネスネットワーキングという明確なインセンティブが属性開示を促進
- **My Opinionへの示唆**: 属性開示に明確なメリット（分析結果の精度向上、信頼性向上）を紐付けることが重要

#### Facebook（実名制の功罪）
- **実名ポリシーの課題**: 苦情ベースの通報システムが悪用され、LGBTQ+、DV被害者、先住民、活動家が不当にアカウント停止される事例が多発
- **2015年にポリシー緩和**: 「特別な事情」の説明機能を追加したが、根本的問題は未解決
- **日本での利用率**: 19.8%（2024年12月調査）と低迷。日本では実名文化がなじみにくい
- **My Opinionへの示唆**: 「実名」ではなく「属性」の開示に絞ることで、個人特定リスクを下げつつ議論の質を担保する設計が差別化ポイント

#### mixi（日本発・招待制SNSの先駆者）
- 2004年サービス開始、招待制で2010年に会員数2,000万人突破
- 「あしあと」機能等の独自機能で爆発的に普及
- **衰退要因**: Twitter/Facebookへの移行、オープン化の遅れ、モバイルシフトの遅れ
- **My Opinionへの示唆**: 招待制は初期の密度確保に有効だが、適切なタイミングでのオープン化が重要

### 1.3 日本国内の政治議論プラットフォーム

| サービス | 概要 | 規模 |
|----------|------|------|
| **選挙ドットコム** | 日本最大の選挙・政治情報ポータル。候補者情報、政策比較 | 年間1億PV超 |
| **Yahoo!ニュース コメント欄** | 日本最大規模の意見表明の場（事実上）。ニュースへのコメント | 月間数千万コメント（推定） |
| **ポリタスTV / Politas** | 政治ジャーナリズムメディア。動画＋記事 | YouTube登録者数万規模 |

**日本市場の空白**: 「属性を明示した上での継続的な政治議論プラットフォーム」は存在しない。選挙ドットコムは情報提供が主、Yahoo!コメントは匿名で質が低い、X(Twitter)はエコーチェンバーの温床。この空白がMy Opinionの参入機会。

### 1.4 エコーチェンバー可視化の先行事例

- 総務省「令和5年版情報通信白書」がフィルターバブル・エコーチェンバーの問題を公式に指摘
- 「エコーチェンバー可視化システムβ版」が過去に存在したが、2023年5月に提供終了
- JST等の研究機関でSNSにおける確証バイアス軽減の研究が進行中
- **My Opinionへの示唆**: 学術的な裏付けがある領域であり、総務省の問題提起とも合致。社会的意義を訴求しやすい

## 2. ターゲット市場分析

### 2.1 日本国内のSNS市場規模・トレンド（2024-2025年）

#### 市場規模
- 日本国内SNS利用者数: **8,452万人**（2024年末）、2026年末に**8,550万人**見込み
- ネットユーザーに占めるSNS利用率: **79.0%**（2024年末）、80.1%（2026年末見込み）
- ソーシャルメディアアナリティクス市場（日本）: **$7.59億**（2024年）、2033年に$50.7億見込み（CAGR 21.7%）

#### 主要トレンド（2024-2025年）
1. **新興SNSの台頭**: Threadsが1,230万ユーザー突破、BeRealが937%成長。「リアル」「気軽さ」への価値シフト
2. **主要SNSの利用率低下**: YouTube(-7.0pt)、X(-5.7pt)の落ち込み。既存SNSへの飽和・不満
3. **プライバシー意識の高まり**: 「個人情報の流出」が最大の懸念（60.9%）
4. **政治×SNSの結合**: 2024年衆院選でSNSが選挙に決定的影響（石丸現象、国民民主党の躍進、兵庫県知事選）
5. **エコーチェンバーへの社会的問題意識**: 総務省が情報通信白書で正式に問題提起

#### My Opinionにとっての追い風
- 既存SNSへの不満（偏り、荒らし、質の低下）が高まっている
- 政治×SNSが2024年に社会的に可視化され、「質の高い議論の場」への潜在需要が顕在化
- 「リアル」志向の流れと「属性開示」のコンセプトが合致

### 2.2 政治議論・意見交換プラットフォームの需要

#### 需要の根拠
- 内閣府調査（2024年10月）: 30代で「国民が参加できる場をひろげる」ニーズが最も高い
- NIRA調査（2024年）: 政治家を信頼する人は全体の約20%に留まり、「そのような政党はない」が30%。既存の政治チャネルへの不信感が強い
- Z世代調査（2023年）: 投票意向7割超だが、「自分が投票しても変わらない」63.6%。**無力感の解消**がキーニーズ

#### 潜在市場規模の推定
- 日本のX(旧Twitter)利用者: 約7,090万人
- 政治・社会問題に関心のある層（推定15-20%）: 約1,060万〜1,420万人
- そのうち「質の高い議論の場」を求める層（推定5-10%）: **約53万〜142万人**
- 初期ターゲット（SAM）: **10万〜30万人**

### 3.2 Verified（本人確認）課金の具体設計

#### Verified制度の構成

| | 無料ユーザー | Verified（¥300/月 or ¥3,000/年） |
|---|---|---|
| **認証バッジ** | なし | あり（チェックマーク） |
| **本人確認** | メール認証のみ | 身分証確認（eKYC） |

#### eKYC実装方針
- 初期: TRUSTDOCK等の外部eKYCサービスを利用（月額1万円〜＋従量課金30〜100円/件）
- ¥300/月（年額¥3,000）はeKYCコストをカバーしつつ荒らし抑制の心理的障壁として機能
- Verified × サブスクのセット割引: Standard + Verified = ¥700/月、Pro + Verified = ¥1,180/月

#### データ提供の前提条件
- 全データは**k-匿名化**（k>=5推奨）を適用し、個人特定を不可能にする
- ユーザーには「匿名化統計データの研究利用」への同意を利用規約で取得
- n < 20のセルは非表示（MVPのプライバシー対策と整合）
- 個人データの直接販売は一切行わない

#### 市場規模の参考
- 日本のソーシャルメディアアナリティクス市場: $7.59億（2024年）、CAGR 21.7%で成長中
- 世論調査1回あたりの市場価格: 100万〜500万円（調査会社）
- My Opinionの常時データは「リアルタイム世論調査」として差別化可能

### 3.4 広告モデルの現実的な収益見込み

#### 広告収益の前提
- **ブランドセーフティの壁**: 政治スタンス等のセンシティブ属性を扱うため、一般的なアドネットワークのCPMは低くなりがち
- 日本のディスプレイ広告平均CPM: ¥200〜¥500
- 政治系コンテンツのCPM想定: ¥100〜¥300（30-50%ディスカウント）

#### 広告収益シミュレーション（Phase 2到達時: MAU 10,000）

| 指標 | 値 |
|------|-----|
| MAU | 10,000 |
| DAU/MAU比 | 30%（3,000 DAU） |
| 1人あたりPV/日 | 15 |
| 日間広告インプレッション | 45,000 |
| 有料ユーザー除外（広告非表示） | -15%（38,250 imp） |
| CPM | ¥200 |
| 日間広告収益 | ¥7,650 |
| **月間広告収益** | **約¥230,000** |

#### 推奨する広告戦略（優先順）
1. **直販スポンサー**: テーマ提供型スポンサー（「今週のテーマ presented by XX」）。月額10万〜50万円
2. **コンテキスト広告**: テーマ・内容に連動した広告。属性による出し分けは避ける
3. **一般アドネットワーク**: Google AdSense等。補助的な位置づけ

**結論**: 広告は補助収益（全体の10-20%）と位置づけ、サブスク＋B2Bを主軸にする

## 4. 成長戦略の具体化

### 4.1 Phase別KPIと施策

#### Phase 0: リリース前準備（リリース2〜4週間前）

**目標**: 創業メンバー30〜100人を集め、投稿・議論文化の「型」を作る

| KPI | 目標値 |
|-----|-------|
| 招待メンバー数 | 50〜100人 |
| 属性入力率 | 80%以上 |
| 週間投稿数/人 | 3件以上 |
| テーマ内返信率 | 50%以上（投稿に対して返信がつく割合） |
| 分析閲覧率 | 70%以上 |

**具体的施策**:
1. **シードユーザーリクルーティング**: 創業者の直接ネットワークから、政治・社会問題に関心の高い知人を招待。大学教員、ジャーナリスト、政策関係者を優先
2. **週次テーマ設定（運営主導）**: 時事的な議論テーマを毎週月曜に設定。例: 「選択的夫婦別姓に賛成か反対か」「大学無償化をどう思うか」
3. **オンボーディングガイド**: 属性入力の意義、分析機能の使い方を丁寧に説明する初回チュートリアル
4. **Slack/Discordでのフィードバック収集**: シードユーザーからの改善要望を即座に反映

#### Phase 1: 0 → 1,000人（リリース後1〜3ヶ月）

**目標**: 「投稿→反応→分析」のループが自然に回る状態を作る

| KPI | 目標値 |
|-----|-------|
| MAU | 1,000 |
| D7リテンション | 40%以上 |
| D30リテンション | 25%以上 |
| 投稿→返信率 | 60%以上 |
| 分析閲覧率 | 50%以上 |
| 課金転換率 | 3%以上 |
| NPS | +30以上 |

**具体的施策**:
1. **ウェーブ制招待**: 毎週100〜200人ずつ新規招待を開放。既存ユーザーに招待コード2枚配布
2. **シェアカード機能**: 投稿の分析結果を画像化してX(Twitter)にシェアできる機能。「この投稿、30代男性は70%がGoodだけど、50代女性は60%がBadだった」のような可視化
3. **テーマ主催者制度**: アクティブユーザーにテーマ主催権限を付与。運営負荷の分散と当事者意識の醸成
4. **プッシュ通知最適化**: 返信通知、フォロー通知、テーマ開始通知を適切な頻度で配信
5. **初期メディア露出**: TechCrunch Japan、ITmedia、CNET Japan等のテック系メディアへのプレスリリース
6. **X(Twitter)での話題化**: 創業者・シードユーザーによる定期的な分析結果シェア、メディア掲載のリツイート

#### Phase 2: 1,000 → 10,000人（リリース後3〜12ヶ月）

**目標**: コミュニティの自走と収益基盤の確立

| KPI | 目標値 |
|-----|-------|
| MAU | 10,000 |
| DAU/MAU比 | 25〜30% |

| Verified率 | 10%以上 |

**具体的施策**:
1. **異論ミックス導線**: 反対属性の投稿を少量差し込むスライダー。「いつもと違う視点」を体験させる
2. **テーマ束ね（イベント型）**: 選挙、国会審議、社会的事件に合わせた集中テーマ週間
3. **モデレーション整備**: 通報→警告→制限→凍結のフロー。コミュニティガイドラインの策定
4. **B2Bパイロット**: 大学研究室2〜3箇所とパイロット契約。事例として活用
5. **インフルエンサー連携**: 政治系YouTuber、NewsPicksプロピッカーへの招待・コラボ
6. **SEO/コンテンツマーケ**: 「エコーチェンバーとは」「政治スタンス診断」等のコンテンツで検索流入を獲得
7. **選挙連動キャンペーン**: 国政選挙・地方選挙に合わせた特設テーマ。選挙前後の意見変化を可視化

### 4.3 リテンション施策

#### 短期リテンション（D1〜D7）
1. **即座の成功体験**: 初回投稿後24時間以内にGood/Badと閲覧者属性の簡易分析を表示（n < 20でも「あと○人の閲覧で分析が見られます」と表示）
2. **返信促進**: 新規投稿に対して属性の異なるユーザーへ返信を促すレコメンド通知
3. **デイリーテーマ**: 毎日1つの軽い投票型テーマ（「賛成/反対」だけで参加可能）で習慣化

#### 中期リテンション（D7〜D30）
1. **週次レポート**: 「今週のあなたの分析サマリー」をプッシュ通知＋メールで配信
2. **エコーチェンバー指標の変化通知**: 「あなたのフォロー偏り度が先週より5%改善しました」
3. **テーマカレンダー**: 今後のテーマ予定を公開。関心のあるテーマに事前登録可能

#### 長期リテンション（D30〜）
1. **累積分析**: 投稿履歴に基づく「あなたの意見変遷マップ」（時系列で自分の立ち位置の変化を可視化）
2. **バッジ・実績システム**: 「100投稿達成」「10テーマ参加」「異論ミックス利用者」等の軽いゲーミフィケーション
3. **テーマ主催者への昇格**: アクティブユーザーにテーマ作成権限を付与し、当事者意識を醸成
4. **コミュニティイベント**: 月1回のオンラインディベートイベント（テーマ主催者がファシリテート）

## 付録: 主要リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 属性詐称（嘘の属性入力） | 高 | Verified制度による段階的な信頼性担保。統計的外れ値検知 |
| 荒らし・誹謗中傷 | 高 | 招待制＋Verified＋モデレーション。通報フロー整備 |
| 初期ユーザー不足による分析不成立 | 高 | n < 20非表示ルール。テーマ空間で「小さな村」を成立させる設計 |
| 政治的偏りによる片方の属性ばかり集まる | 中 | Phase 0でバランスのとれたシードユーザー確保。テーマ設計で多様な層を呼び込む |
| 個人情報保護法・プライバシー問題 | 高 | k-匿名化、n < 20非表示、利用目的の明示（別途法務調査で詳細化） |
| 収益化の遅れ | 中 | インフラコストを最小化し、黒字化ハードルを低く設定 |
